# websocket-tool Branding

## Product Name

- Primary name: `websocket-tool`
- Category descriptor: `WebSocket Traffic Inspector`
- Tagline: `Inspect, replay, and control WebSocket traffic.`

## Positioning

websocket-tool is a developer-first debugging extension focused on three jobs:

- seeing every WebSocket frame in context
- replaying client and server messages quickly
- controlling noisy or unstable traffic during debugging

## Tone

- Practical, fast, developer-focused
- Calm under heavy traffic and error conditions
- Clear and precise rather than flashy

## Visual Direction

- Keep the current signal-wave icon language
- Favor deep slate, teal, and electric cyan accents
- Avoid “official upstream fork” cues in screenshots and listings

## Naming Rules

- Use `websocket-tool` in UI titles, store copy, and docs
- Use `websocket-tool` rather than `WebSocket DevTools` in visible labels
- Keep upstream copyright only in `LICENSE`
